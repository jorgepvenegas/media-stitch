Proposed Component Boundaries

How would you split up the app?

A

Feature-based (3 panels)

PreviewPanel — video player + timestamp panel
SyncPanel — offset input, queue, apply button
TimelinePanel — sidebar, timeline SVG, axis, details

Each panel manages its own sub-state. Shared state via Pinia.

B

Granular / file-centric

PreviewPlayer — video or image display
TimestampPanel — copy ISO, use as ref
SyncControls — offset input + action buttons
OffsetQueue — pending stack display
SidebarList + SidebarItem — file list
TimelineView — SVG rendering + axis
FileDetails — details panel
RenderModal + ApplyModal

More files, but each component stays small and focused.

C

Two-view with router

App.vue — layout shell
SyncView — sync panel + file list (full width)
TimelineView — preview + timeline (full width)
Vue Router — switch between sync ↔ timeline modes

Different workflow, not all panels visible at once.

My recommendation: A (Feature-based)
Fits the current layout well, reasonable number of components, keeps related logic together.
